# Qwen3.5 数学推理后训练实验执行计划（适配1-2天完成，双A6000可选并行）
## 一、实验基础信息
1.  模型选型：Qwen3.5-4B（稠密版，兼顾性能与训练速度，A6000单卡/双卡均可适配）
2.  硬件配置：单A6000（必选）+ 双A6000并行（可选加速，非强制）
3.  实验周期：24-48小时内完成全流程
4.  核心流程：数据筛选→SFT→GRPO强化学习→多Benchmark评估→消融分析

## 二、前置准备阶段
1.  环境配置
    - 基于CUDA 12.1+PyTorch 2.3，安装Transformers、PEFT、TRL、Datasets、Accelerate
        (已有环境conda mathRL)
    - 配置Accelerate，预留单卡/双卡训练切换配置文件
2.  模型与数据下载
    - 下载Qwen3.5-4B基座模型至本地按照plan2.md完成任务
    - 拉取DeepMath-103K、Big-Math-RL-Verified数据集，预留MATH-500/GSM8K/TheoremQA评估集
    （已有数据集和评估集 /mathRL/data）
3.  方案切换准备
    - 编写单卡/双卡训练参数分支，双卡仅作为加速可选项，不影响单卡执行

## 三、数据构建与筛选阶段
1.  多源数据合并：整合DeepMath与Big-Math-RL数据集
2.  标准化清洗：长度过滤、乱码清洗、LaTeX答案规范化、哈希去重
3.  格式转换：转换为Alpaca指令格式与GRPO专用Prompt格式
4.  数据集划分：95:5划分训练集与验证集，精简数据集规模适配短周期实验

## 四、LoRA-SFT训练阶段
### 核心参数（单卡默认，双卡可选）
1.  基础配置：bf16精度，开启梯度检查点，序列长度4096
2.  LoRA参数：r=64，lora_alpha=128，lora_dropout=0.05，适配Qwen3.5目标模块
3.  训练参数：单卡batch_size=8，累积步数4；双卡开启FSDP/DP并行，总batch翻倍
4.  优化器：paged_adamw_8bit，学习率2e-4，warmup步数50，控制总步数适配周期
5.  执行逻辑：单卡直接运行，双卡切换Accelerate配置启动，输出SFT LoRA权重

## 五、GRPO强化学习阶段
### 核心参数（适配Qwen3.5，短周期收敛）
1.  生成配置：n_candidates=4，max_new_tokens=1024，温度0.7
2.  奖励函数：正确性奖励+格式奖励复合权重，适配Qwen3.5生成范式
3.  训练配置：单卡保证显存≤42GB，双卡并行提升训练效率，减少迭代时间
4.  收敛控制：提前设置reward阈值停止条件，避免过度训练耗时
5.  执行逻辑：加载SFT模型启动GRPO，双卡为可选项，单卡可独立完成

## 六、统一评估与分析阶段
1.  评估模型：Qwen3.5-4B基座、SFT模型、GRPO模型
2.  评估基准：GSM8K、MATH-500、TheoremQA，自动化提取boxed答案判分
3.  加速方案：使用vLLM加速推理，单卡可完成，双卡可并行评估不同基准
4.  结果输出：准确率对比表、训练曲线、数据清洗漏斗图
5.  失效分析：统计reward hacking、格式偏移、计算错误等问题

## 七、消融实验
1.  精简消融：仅做奖励权重消融、单双卡训练效果对比
2.  快速验证：小批量数据集验证参数有效性，保障1-2天周期内完成

## 八、时间总控与切换规则
1.  总时长：单卡约22-29小时，双卡可选加速至18-24小时
2.  切换规则：训练前通过Accelerate配置选择单/双卡，不修改核心训练逻辑
3.  风险控制：精简数据集与训练步数，优先保障主流程完成，消融实验按需取舍
